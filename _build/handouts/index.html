Calculator

Introduction

This exercise will have you complete functions to finish implementing a prefix calculator CLI application (an application that runs in the terminal).

The calculator lets a user add, subtract, multiply, divide, square a number, cube a number, raise a number to a given power, and find the remainder when one number is divided by another (otherwise known as taking one number mod another).

Setup

Download the materials for this exercise here.

Once the materials have finished downloading, cd into the project directory. Then, initialize a Git repo, git add all relevant files, and make an initial commit.

Introduction

Calculators and Notation

We typically write math equations using infix notation like this — 3 + 2 — where the operator (in this case, the plus sign) goes inside the numbers.

Another way to write equations is with prefix notation. As the name implies, in prefix notation, operators go before the operands so, instead of 3 + 2, you’d write + 3 2.

One advantage of prefix notation is the ability to have an arbitrary number of operands. For example, 4 + 3 + 2 in prefix notation is + 4 3 2.

In this exercise, we will build a basic prefix calculator together. We will provide half of the program, and you will code the other half. We will use this idea to explore the concept of an application programming interface.

Recall that a function has three key parts:

  • The name of the function

  • Its parameters or input

  • What the function does and/or what it outputs

Together, these three pieces of information are a functions’s signature.

A function signature is a way of describing a function without going into the details of how it operates.

Completing Functions for a Calculator App

How the App Works

We’ve provided a terminal-based prefix calculator app with your exercise materials as a Java file called Calculator.java. It relies on the other file we’ve included — calc/Arithmetic.java.

First, let’s get oriented by compiling and running Calculator.java in your preferred command line. When the app starts, try adding 2 numbers together:

 $ javac Calculator.java
 $ java Calculator
Enter your equation >
+ 1 2
10.0

As you can see, Calculator.java doesn’t work — + 1 2 is definitely not 10!

This happens because Calculator.java relies on functions from Arithmetic.java and the problem with Arithmetic.java is… well… you should check it out and see for yourself. Why do you think + 1 2 gave 10 instead of 3?

The problem with Arithmetic.java

All the methods in this file return 10.0.

Finish Arithmetic.java

Your task is to complete the functions in Arithmetic.java so Calculator.java will actually work. Here’s an example of what the calculator should be able to do:

 $ java Calculator
Enter your equation > + 1 2
3.0
Enter your equation > - 10 5
5.0
Enter your equation > * 2 3
6.0
Enter your equation > / 7 2
3.5
Enter your equation > square 2
4.0
Enter your equation > cube 3
27.0
Enter your equation > pow 2 5
32.0
Enter your equation > mod 10 3
1.0
Enter your equation > q

You can start by fixing the add function so + 1 2 prints the proper value to the screen. When you’re finished with add, run Calculator.java so you can test and double-check that your function works:

 $ javac Calculator.java
 $ java Calculator
Enter your equation > + 1 2
3.0
Enter your equation > + 100 2
102.0

Then — since you’ve just finished a unit of working code — you should stage and commit your changes:

$ git add Arithmetic.java
$ git commit -am "Completed add function"

Rinse and repeat for the rest of the functions in Arithmetic.java.

Stop here and complete the functions in Arithmetic.java. Move on to the next section when you’re done.
